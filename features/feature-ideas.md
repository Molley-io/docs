## Prioritization

### Priority Levels

| Priority | Description | Action |
|----------|-------------|--------|
| **Critical** | Must have, blocking | Do immediately |
| **High** | Important, significant value | Do soon |
| **Medium** | Valuable, not urgent | Plan for later |
| **Low** | Nice to have | Consider when time allows |

### Evaluation Criteria

Consider these factors:

- **User value** â€” How much does this help users?
- **Business value** â€” How does this help the business?
- **Effort** â€” How much work is required?
- **Risk** â€” What could go wrong?
- **Dependencies** â€” What else is needed?

## Moving Ideas Forward

### To Consideration

When an idea is worth evaluating:

1. Open the idea
2. Change status to **In Consideration**
3. Add evaluation notes
4. Discuss with team

### To Refinement

When approved for development:

1. Change status to **In Refinement**
2. Add detailed requirements
3. Create specifications in Product Wiki
4. Estimate effort

### To In Progress

When development starts:

1. Change status to **In Progress**
2. Track in Feature Kanban
3. Update requirements as work progresses

### To Released

When shipped:

1. Change status to **Released**
2. Add release notes
