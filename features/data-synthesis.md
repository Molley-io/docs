# Data Synthesis

Data Synthesis helps you analyze research data and transform it into actionable insights using AI.

## Overview

Data Synthesis is designed for:

- **Competitive analysis** — Compare competitors systematically
- **User research synthesis** — Find patterns across interviews
- **Market analysis** — Understand industry trends
- **Feature analysis** — Evaluate feature opportunities

## Creating an Analysis

### New Analysis

1. Navigate to **Research** → **Data Synthesis**
2. Click **+ New Analysis**
3. Choose an analysis type:
   - Competitive Analysis
   - User Research
   - Market Analysis
   - Feature Analysis
4. Enter a title and description

### Analysis Types

#### Competitive Analysis

Compare your product against competitors:

- Feature comparison matrices
- Pricing analysis
- Positioning maps
- Strengths and weaknesses

#### User Research

Synthesize findings from user interviews:

- Pattern identification
- Pain point clustering
- Need prioritization
- Persona refinement

#### Market Analysis

Understand your market landscape:

- Trend identification
- Market sizing
- Opportunity mapping
- Risk assessment

#### Feature Analysis

Evaluate feature opportunities:

- Impact vs effort matrices
- User value assessment
- Technical feasibility
- Priority recommendations

## Working with Analyses

### Adding Data Sources

Link research to your analysis:

1. Open your analysis
2. Click **+ Add Source**
3. Select from:
   - Research Notes
   - Knowledge Notebooks
   - External links
   - Manual input

### AI-Powered Synthesis

Once you've added sources:

1. Click **Synthesize with AI**
2. Molley analyzes all connected sources
3. Generates insights, patterns, and recommendations
4. Review and refine the output

### Manual Editing

Combine AI synthesis with your expertise:

- Edit generated content
- Add your own insights
- Reorganize sections
- Include custom visualizations

## AI Context

When synthesizing data, Molley AI considers:

- Your company profile and goals
- Your project's target audience
- Existing features and roadmap
- Industry context

This means insights are tailored to your specific situation, not generic recommendations.

## Best Practices

### 1. Quality Sources

Better input = better output:

- Use detailed research notes
- Include diverse perspectives
- Add quantitative data when available

### 2. Iterative Refinement

Don't expect perfection on first try:

- Review AI output critically
- Add your domain expertise
- Re-synthesize with additional context

### 3. Connect to Action

Link insights to outcomes:

- Create feature ideas from findings
- Update your product wiki
- Inform marketing strategy

### 4. Regular Updates

Keep analyses current:

- Add new research as it comes in
- Re-run synthesis periodically
- Archive outdated analyses

## Example Workflow

### Competitive Analysis Example

1. **Gather sources**
   - Competitor websites
   - Product reviews
   - Industry reports
   - User feedback

2. **Create analysis**
   - Select "Competitive Analysis"
   - Link all sources
   - Add specific questions to answer

3. **Run synthesis**
   - AI analyzes all sources
   - Generates comparison matrix
   - Identifies gaps and opportunities

4. **Refine and act**
   - Review AI findings
   - Add your insights
   - Create feature ideas from gaps
   - Update product strategy
